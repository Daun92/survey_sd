# 06. UX 라이팅 · 메시지 시스템

> **핵심 원칙.** 카피는 UI 의 장식이 아니라 *UX 의 뼈대* 다 (토스 UX 라이팅 내재화). 한국어 문자열은 컴포넌트에 하드코딩하지 않고, 의미 단위로 카탈로그에 모은다. 에러·빈 상태·CTA 는 규칙 있는 구조로 쓴다.

### 문제

- 유사 문구가 서로 다른 단어로 흩어져 있어 QA·번역·브랜드 톤 일관성 유지 불가.
- 에러 카피가 "무엇을 하면 되나" 를 말해주지 않는다 (예시 1, 2).
- 성공/완료 카피가 "감사합니다" 처럼 빈 인사로 끝난다 (예시 7).
- `06` 이 책임질 카탈로그 인프라가 아직 없다.

## 설계 원칙

### A. 메시지 카탈로그 구조

```
src/messages/
└── ko/
    ├── index.ts              # t() 진입점, 타입 안전한 키 union 발급
    ├── admin.ts              # /admin/*
    ├── respondent.ts         # /s, /d, /hrd
    ├── error.ts              # 공통 에러 / fallback
    ├── empty.ts              # EmptyState 카피
    ├── auth.ts               # 로그인·권한
    └── shared.ts             # 공통 단어(저장·취소·삭제·다음·이전·제출)
```

- 번들 단위로 분리해 트리셰이킹 + 유지 관리 명확성 확보.
- 지금 단계에서 i18n 라이브러리는 도입하지 않는다 (MoSCoW: Won't in P1/P2). 단순 `t(key, vars?)` 함수 + 문자열 보간만 사용 → 추후 `next-intl` 등으로 확장 가능한 **호환 가능 구조** 만 유지.

### B. 톤·보이스 매트릭스

| 맥락 | 톤 | 보이스 | 사람 비유 |
|---|---|---|---|
| 응답자 안내 | 존댓말 | 친근·배려 | 안내 데스크 직원 |
| 응답자 에러 | 존댓말 | 담담·해결 지향 | 침착한 기술 지원 |
| 관리자 일반 | 평서형 간결 | 도구적 | 숙련된 사수 |
| 관리자 파괴 액션 | 평서형 | 경고·명확 | 규정을 말하는 관리자 |
| 마케팅 (추후) | — | — | 이 디자인 시스템 스코프 밖 |

### C. 카피 3요소 (에러·경고)

모든 에러·경고 카피는 다음 3요소를 포함해야 한다.

1. **무슨 일** — 사용자 입장에서 관찰한 사실.
2. **왜** (생략 가능) — 사용자가 행동으로 연결할 수 있는 원인.
3. **다음 행동** — 지금 할 수 있는 단 하나의 동작.

```
❌ "오류가 발생했습니다."
✅ "응답을 저장하지 못했어요. 네트워크가 잠시 끊긴 것 같아요. 다시 시도 버튼을 눌러 주세요."
```

### D. CTA 라이팅 규칙

- 동사-중심 (`저장`, `보내기`, `응답 제출`) — 명사-중심(`완료`)은 지양.
- 파괴 액션 = 동사 그대로 (`설문 삭제`, `차수 삭제`). "정말?" 같은 수식 금지.
- 응답자 경로의 `다음`/`이전` 은 예외적으로 **단어 유지** (엄지영역 인지 속도 우선).

### E. 빈 상태 (EmptyState) 라이팅

- 타이틀: "아직 없는 것" 을 1줄로 (`응답이 아직 없어요`).
- 본문: "어떻게 생기는가" 를 1줄로 (`응답자가 링크에 접속해 설문을 제출하면 이곳에 표시돼요.`).
- 선택적 CTA: "그래서 무엇을 해야 하는가" (`배포 링크 열기`).

### F. 확인 다이얼로그 라이팅

- 타이틀: "무엇을 한다" (명령형, 조사 없음 — `설문 삭제`).
- 본문: 결과 + 복구 가능 여부 (`이 차수와 응답자 12명의 링크가 삭제돼요. 되돌릴 수 없어요.`).
- 버튼: `취소` / `삭제` — 영어 `Cancel/Delete` 와 1:1.

### G. i18n 준비 레벨

- Phase 1: 상수화만. `t('respondent.inactive.title')` 수준.
- Phase 2: 보간 지원 (`t('distribute.status.failedBatch', { count })`).
- Phase 3: 라이브러리 도입 검토. 지금은 키·구조만 호환되면 된다.

| `src/messages/ko/` skeleton + 타입 | Must | P1 | 신규 |

## 수용지표

**정량**
- 사용자 노출 한국어 문자열(현 grep 기준) 중 카탈로그 경유 비율: Phase 1 ≥ 50% (루트·응답자 상태), Phase 2 ≥ 80%.
- "오류가 발생했습니다" 류 1문장 에러 메시지 수: 현재 여러 곳 → 0.
- PR 템플릿 카피 체크리스트 통과율: 100% (리뷰어 반려 없이 통과한 PR 비율).

**정성**
- 응답자가 "왜 이런 화면인지" 알 수 있고 다음 행동을 알 수 있다 (3요소 기준 리뷰 합격).
- 같은 상태가 다른 페이지에서 다른 말로 쓰이지 않는다.

## 체크리스트

- [ ] 카피를 컴포넌트에 직접 쓰지 않고 `src/messages/ko/*` 에 뒀다.
- [ ] 에러·경고 카피가 3요소(무슨 일·왜·다음 행동)를 지킨다.
- [ ] CTA 가 동사-중심이다.
- [ ] 같은 의미의 기존 키가 있는지 확인했다.
- [ ] 파괴 액션 카피가 결과·복구 가능 여부를 말한다.
- [ ] 톤 매트릭스의 해당 칸과 일치한다.
